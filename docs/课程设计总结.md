# 课程设计总结

## 一、设计题目

基于 Grad-CAM 的深度学习模型可解释性可视化系统

## 二、设计目的

1. 理解卷积神经网络、视觉 Transformer 的基本工作原理
2. 掌握 Grad-CAM、Grad-CAM++ 算法原理与代码实现
3. 学习 PyQt5 桌面界面开发
4. 熟悉 PyTorch 模型调用、图像预处理、梯度计算等全流程
5. 对比不同算法的可视化效果，分析算法优劣

## 三、主要完成工作

### 1. 环境搭建

使用 Anaconda 创建虚拟环境，安装 PyTorch、PyQt5、OpenCV 等依赖库，

解决版本冲突、包安装错误等问题。

### 2. 算法开发

基于开源 grad-cam 库二次开发，实现两种主流可解释性算法，

同时兼容 ResNet50、VGG16、ViT-B/16 三类模型。

### 3. 界面开发

使用 PyQt5 搭建完整桌面程序：

- 划分基础功能、进阶功能两大模块
- 实现按钮、布局、状态提示、弹窗交互
- 完成多页面切换、多线程处理

### 4. 功能调试

修复梯度丢失、程序卡死、进程闪退、算法结果异常等问题，

保证批量、视频、摄像头、多算法对比全部功能稳定运行。

### 5. 文档编写

完成项目说明书、原理解析、实验记录、总结全套文档。

## 四、遇到的问题与解决方案

1. 模型 eval 模式无法计算梯度

   解决方案：计算热力图时临时切换为 train 模式，执行后恢复。

2. 视频、摄像头运行程序卡死

   解决方案：使用 Qt 子线程分离耗时操作，增加资源释放逻辑。

3. 热力图叠加后原图变色

   解决方案：改用 OpenCV 加权叠加，保留原始图像色彩。

4. 第三方库安装名称混淆

   解决方案：区分安装名 grad-cam 与代码导入名 pytorch_grad_cam。

## 五、实验结果分析

1. ResNet50、VGG16 卷积网络：两种算法均可准确定位目标。
2. Grad-CAM：关注区域偏大，易包含少量背景。
3. Grad-CAM++：定位更精细，目标边缘效果更好。
4. ViT 模型：算法可正常适配，注意力分布与卷积网络存在差异。
